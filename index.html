How I Built and Deployed My Professional CV Website on AWS with Terraform & GitHub Actions

Author: Ibrahim Kilicaslan
Date: [Add your publish date]

Introduction

As a Cloud & DevOps Engineer, I wanted my personal website to reflect not only my professional experience but also my technical skills in automation, cloud infrastructure, and best practices. Instead of using a simple website builder, I decided to build and deploy my CV website using AWS services (S3, Route 53, CloudFront, ACM) and automate everything with Terraform and GitHub Actions.

In this post, I'll share my journey, the architecture, the challenges I faced, and a step-by-step guide for anyone who wants to build a modern, secure, and fully automated static website on AWS.

Why Not Just Use a Website Builder?

  • Full control over infrastructure and deployment
  • Learning and showcasing real-world DevOps and cloud skills
  • Scalability and security with AWS best practices
  • Automation: No manual uploads, no FTP, no "click and pray" deployments

Architecture Overview

The website is a static HTML/CSS/JS site, hosted on AWS S3, delivered globally via CloudFront, secured with HTTPS (ACM), and managed DNS with Route 53. All infrastructure is defined as code with Terraform, and deployments are automated with GitHub Actions.

  • S3: Static website hosting
  • CloudFront: CDN, HTTPS, custom domains
  • Route 53: DNS management
  • ACM: Free SSL certificates (auto-renew)
  • GitHub Actions: CI/CD for automatic deployment

Step-by-Step: How I Did It

1. Design & Frontend

  • Built a modern, responsive CV website using HTML, CSS, and JavaScript.
  • Modularized sections for easy editing (e.g., education, experience, projects).

2. Infrastructure as Code with Terraform

  • Created a modular Terraform setup: S3 bucket, CloudFront, Route 53, ACM certificate
  • All variables (domain, subdomains, region) are in variables.tf for easy customization.

3. CI/CD with GitHub Actions

  • Set up a workflow that syncs all website files to S3 on every push to main
  • Invalidates CloudFront cache so changes are live instantly
  • AWS credentials and CloudFront Distribution ID are stored as GitHub Secrets for security.

4. Domain & DNS

  • Registered my domain (ibrahimkilicaslan.click)
  • Managed all DNS records via Terraform/Route 53
  • Updated registrar NS records to point to Route 53

5. SSL & Security

  • Used ACM for a wildcard SSL certificate (auto-renew, no manual steps)
  • Enforced HTTPS via CloudFront
  • S3 bucket is public only for static site content, not for uploads or sensitive data

What Makes This Project Unique?

  • Everything is code: No manual AWS Console steps after initial setup
  • Professional documentation: Step-by-step README, troubleshooting, and customization tips
  • Beginner-friendly: Anyone can fork, edit, and deploy their own site with minimal AWS knowledge
  • Scalable and secure: Uses AWS best practices for static sites

Challenges & Solutions

  • ACM DNS validation: Required a two-step Terraform apply (first create cert, then validate)
  • S3 bucket name conflicts: Used unique domain-based names
  • CloudFront not updating: Solved with automatic cache invalidation in CI/CD
  • DNS propagation delays: Documented troubleshooting steps for users

How You Can Build Your Own

  1. Fork or clone my repo: GitHub - ibrahimkilicaslan-website
  2. Edit your content: Update index.html, css/styles.css, and other assets
  3. Customize your domain: Edit terraform-static-website/variables.tf
  4. Deploy infrastructure: Run terraform init and terraform apply (see README for details)
  5. Set up GitHub Secrets: Add your AWS keys and CloudFront Distribution ID
  6. Push your changes: Every push to main will auto-deploy your site to AWS!
